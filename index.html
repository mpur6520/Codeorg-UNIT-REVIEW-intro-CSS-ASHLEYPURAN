Lesson 9: Lossless Compression

The point of this lesson was to learn about lossless compression, a way where a computer can shrink down data without losing any of it.

What is the most important quality of lossless compression?
The data is transformed to usually make it smaller. It can always be reconstructed back to the original.
An author is preparing to send their book to a publisher as an email attachment. The file on their computer is 1000 bytes. When they attach the file to their email, it shows as 750 bytes. The author gets very upset because they are concerned that part of their book has been deleted by the email address.

If you could talk to this author, how would you explain what is happening to their book?
I would say that there isn't a part of their book being deleted, but rather that the file was compressed. I'd explain that they most likely have repeated words such as "I" or another vocabulary that, within code, can be categorized as data being grouped together--these words aren't lost and the action can be undone, but it's just done for the file size to be smaller.

Lesson 10: Lossy Compression

The point of this lesson was to learn about lossy compression, a way where a computer can shrink down data but lose some in the process; we learned about scenarios where this is a good method.

You’ve been given a new cell phone with a 2-gigabyte data plan. You plan to use your phone for text messages, images, videos, and music. Which of these categories are best compressed using lossless compression? Which of these categories are best compressed using lossy compression? Why?
I think the categories that are best compressed using lossless compression include text messages and music, while the best for lossy compression would be videos and images. This is because I think that you wouldn't be able to notice a loss of data with videos or images as much as you would with text messages or music--I think the music would become choppy, and the text messages would be harder to read or understand.
